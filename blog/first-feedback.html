First feedback

I view feedback as the main source of growth. I prefer starting by building something my way, then request review and make changes where it is needed. Besides the highlighting of obvious code mistakes I especially seek feedback that changes and upgrades my thinking and designing process. Mentioning that, my first feedback on this project was that i overly heavy on some minor things in the begining, leading to me having practically several days to complete most of the website. Of course, this was a mistake, and a better way would be to plan firstly, then deliver an MVP, and only after add that gimmicky features if there would be some time left.
